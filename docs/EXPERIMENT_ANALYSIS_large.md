# Indexing Experiment — Large Dataset Analysis

This analysis interprets the results from `indexing_experiment_report_large.txt` generated by:

```bash
python3 src/indexing_experiment.py --db data/main_V2.db --repeats 200 --report indexing_experiment_report_large.txt
```

## How to Read the Report
- "No Index (ms)" vs "Indexed (ms)": timing across repeated runs
- "Execution Plan Analysis": watch for shifts from TABLE SCAN to INDEX SEARCH
- "Summary": average improvement and speedup factor

## Expected Differences vs Small Dataset
- Larger tables: indexes usually provide larger speedups (often 3–10x+)
- Complex filters (OR across tables) may still show limited improvement
- Full-list queries without WHERE may improve modestly due to JOIN index usage
